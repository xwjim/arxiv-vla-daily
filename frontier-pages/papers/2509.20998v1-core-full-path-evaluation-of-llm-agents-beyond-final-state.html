一、 📌 论文元数据

  • 标题： CORE: 超越终局状态，对LLM Agent进行全路径评估
  • 核心机构： Synkrasis Labs, Harbin Institute of Technology
  • 一句话本质： 提出用 确定有限自动机(DFA) 评估Agent的全路径行为，而非仅看最终结果。

二、 🎯 破局点 (The Gap)

  • 已有共识/做法： 现有Agent评测大多只看任务 最终状态是否正确，进行“成功/失败”的二元判断。
  • 核心痛点： 这种“唯结果论”忽略了执行过程中的 安全性、效率和中间步骤的正确性，无法区分“侥幸做对”和“完美执行”。
  • 本文切入点： 作者认为，任务可以被抽象为一张“地图”（DFA），Agent的行动就是在这张地图上的“行走路径”。通过对比实际路径和标准路径，可以更全面地评估Agent。

三、 ⚙️ 核心机制 (How it works)

  • 输入 -> 输出： Agent的原始 函数调用序列 -> 一套包含 正确性、安全性、效率 的多维度评估分数。

| 关键模块 | 核心作用 | | :--- | :--- | | 任务的DFA建模 | 将一个任务（如“给植物浇水”）抽象成一个状态机，定义了所有 合法的、安全的 操作步骤和顺序。 | | CORE指标套件 | 基于DFA，从五个维度量化Agent的执行路径与“黄金路径”之间的差异。 | | 路径压缩 (Condensation) | 在评估前，先过滤掉 无状态改变的重复操作（如反复读取），聚焦于真正改变世界状态的关键步骤。 |

四、 💡 核心认知与发现 (Key Insights)

  • [评估盲区]： 终局正确的Agent，其执行路径可能充满风险和低效。 传统评估会给高分，但CORE能识别出那些“跳过必要检查”、“执行危险操作后又撤销”或“疯狂试错”的伪高分Agent。
  • [错误归因]： Agent的“犯错”是多维度的，需要解耦分析。 CORE的五项指标揭示了不同Agent的“性格”：
  • 有的Agent 顺序感差 (PC-KTC低)，总把步骤搞反。
  • 有的Agent 爱走弯路 (Efficiency低)，执行大量冗余操作。
  • 有的Agent 鲁莽冒进 (Prefix Criticality低)，总在任务早期就犯下致命错误。
  • [性能画像]： 模型能力不仅是“能不能做”，更是“做得好不好”。 实验表明，更强的模型（如GPT-4o-mini）不仅终局正确率高，其执行路径的 正确性(PC)、效率和安全性 也全面领先。这为选择部署在真实世界（如机器人、物联网）的Agent提供了更可靠的依据。

五、 🧐 致命弱点与演进方向 (Critical Analysis)

🛑 核心局限 (Limitations)：

  • [抽象瓶颈] DFA模型 表达能力有限，难以描述具有 连续状态、复杂并发或随机性 的真实世界任务。
  • [人力成本] 为每个任务 手动构建DFA 是一个巨大的工程，限制了该评估框架的规模化应用。

⚠️ 论证瑕疵：

  • 论文中多个关键指标（如PC-KTC的权重λ，Prefix Criticality的基数β）的 超参数选择较为随意（如设为0.5），缺乏对其敏感性和影响的深入分析。

🚀 下一步高价值方向 (Next Steps)：

  • [自动化DFA构建] 研究如何从任务描述、代码或演示中 自动学习或生成DFA，这是将CORE框架实用化的关键。
  • [在线路径优化] 将路径评估信号用作 实时反馈，在线指导或修正Agent的行为，甚至作为强化学习的 奖励函数，训练出更安全高效的Agent。
